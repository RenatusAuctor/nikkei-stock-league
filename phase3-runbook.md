## 2. 매수 전 체크리스트 (매수일 T 기준)

- [ ] **T-7~T-3: 결산 캘린더 확인**(§3) — 매수일이 보유 예정 종목의 결산 발표 직전/당일이 아닌지. 체결이 당일 종가라 발표일 매수는 갭 리스크.
- [ ] **T-2~T-1: 47사 시총 재확보** — irbank 표시값 인용 + "주가×주식수=시총" 정합 체크(의심 종목은 닛케이 교차검증). **주식분할·병합 이력 확인 필수**(§8.4에서 닛토방적 분할 미반영 오표시 전례).
- [ ] `node rescore.js new-caps.json` 실행 → 출력으로 `phase2-data.js`의 mktcapNow·popNow·vNow·qarpNow·rankNow 갱신 (mktcapNowAt·rescoredAt도).
- [ ] `node allocate.js` 실행 → 최신 qarpNow 기준 목표 금액표 산출.
- [ ] **T일: 주수 계산** — 주수 = floor(목표금액 ÷ 전영업일 종가). 고가주(디스코)는 1주 단위로 목표금액에 근사. 합계(매수금액×1.011)가 500만엔 이하인지 확인.
- [ ] 주문 입력 → 다음날 새벽 2시 전 내용 재확인(취소 가능 시한).
- [ ] 매수 완료 후: 체결가·주수·수수료를 기록(리포트 부록 + §6 추적표 기준선).

## 4. 비중 방법론 (확정) + 드라이런

**산식**: ① 기본 비중 ∝ qarpNow(현재가 기준 QARP — 테마 게이트를 통과한 14사 *안에서* "좋고 싸게") ② **상한 10% / 하한 4%** 클립 후 잔여 비례 재배분 ③ 예산 500만엔에서 매수 수수료 1.1% 차감. 구현: `allocate.js`.

**설계 논거(리포트에 그대로 쓸 것)**:
- 상한 10% — 단일 종목 의존 방지(분산투자 심사축 정면 대응).
- 하한 4% — 테마 논증에 필수인 종목(도호쿠 2사·시바우라 등 현재가 순위 하위)이 **형식적 편입이 되지 않게 보장**. "규슈 12+도호쿠 2" 분산이 실질이 되는 장치.
- qarpNow 비례 — 선정(앵커 게이트)과 비중(밸류 규율)의 역할 분리. 비싸진 종목(디스코·닛토방적)은 빼는 게 아니라 **가볍게 든다**.

## 5. 데이터 갱신 파이프라인 (요약)

```
① 47사 시총 수집(irbank 표시값 + 주가×주식수 정합 체크, 분할 확인)
      ↓  new-caps.json  {"6855": 1200, ...} ← 47사 전량(부분 갱신 금지 — 백분위 오염)
② node rescore.js new-caps.json   → v·qarp·rank 재산출 표
      ↓  수동 반영(phase2-data.js: mktcapNow·popNow·vNow·qarpNow·rankNow·meta 날짜)
③ node rescore.js                 → 재현 검증(불일치 0 확인)
④ node allocate.js                → 비중·목표금액표
```
q(품질)는 고정 — 니치 점유율 1차 검증 완료(plan §9.3), 재무는 결산마다 FIGURES 갱신 시에만 변동.

## 6. 벤치마크·성과 추적 (리포트 서사용 — 평가액은 심사 무관)

- 기준선: 매수일의 포트폴리오 취득원가(수수료 포함)와 **닛케이평균 / TOPIX / (참고)SOX지수** 종가를 같은 날 기록.
- 이후 **주 1회(금요일 종가)** 4개 값을 기록 — 리포트의 "장기보유 관찰" 섹션 재료. 반도체 사이클 하강이 오면 오히려 "테마 내 분산(밸류체인 단계별 민감도 차이)"이 작동하는지 관찰하는 서사로 활용.
- 결산 발표(§3) 후 각사 실적 한 줄 메모 — 필드워크 질문거리로 연결.

## 7. 리셋 카드 (2회, ~11/30) 사용 기준

원칙: **사용하지 않는 것이 기본**(장기보유 서사와 일관). 예외적으로 고려할 상황:
1. 보유 종목의 **앵커 인과가 무너지는 공시**(예: 구마모토 거점 철수·매각) — 선정 논리 자체의 훼손이므로 교체 사유.
2. 정리·감리종목 지정 등 규정상 거래 제약.
3. Q1 실적으로 **재무 게이트(OPM 8%·자기자본 40%) 명백 이탈 + 구조적 사유** — 단, 일시 요인이면 유지(게이트는 통기 기준).
가격 하락만으로는 리셋하지 않는다(심사는 평가액 무관 — 오히려 "왜 유지했나"가 서사가 된다).
